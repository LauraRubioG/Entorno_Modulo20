<h2 style="color: #ffafcc;">Project Overview</h2>

This project was developed as part of the <b>Assignment: AI-Assisted Development and Git Mastery</b>. The main objective was not only to create a functional and aesthetically pleasing application but also to demonstrate proficiency in utilizing Generative AI as a collaborative programming partner and mastering version control through the Git Command Line Interface.

<h2 style="color: #ffafcc;">Phase 1: The AI-Pair Program</h2>

For this phase, I conceptualized and built the <b>Furry Monster Creator</b>. This application allows users to visually assemble a creature by toggling various accessories, selecting limb types, and adjusting the core color while preserving the original artwork's texture.

<b>Key Features:</b>
<ul>
  <li><b>Interactive UI:</b> A start screen with seamless transitions to the main laboratory.</li>
  <li><b>Dynamic Customization:</b> Users can toggle horns, wings, and tails, as well as choose between different arm and leg types.</li>
  <li><b>Hue Rotation:</b> A specialized color slider that alters the base color of the monster without losing the intricate furry shading.</li>
  <li><b>Adoption System:</b> A generated adoption sequence with randomized pricing.</li>
</ul>

Throughout the development process, an AI assistant was utilized to draft complex logic, structure the DOM manipulation, and refine the CSS positioning to ensure all graphical assets aligned flawlessly.

<h2 style="color: #ffafcc;">Phase 2: Git Command Line Integration</h2>

This repository serves as a testament to strict version control practices managed entirely via the integrated terminal. 

<b>Workflow Highlights:</b>
<ul>
  <li><b>Initialization:</b> The project began with a clean <code>git init</code> environment.</li>
  <li><b>Staging and Committing:</b> Consistent use of <code>git add</code> and <code>git commit -m</code> to track logical milestones in the application's growth.</li>
  <li><b>Branch Management:</b> Features and major styling overhauls were isolated using branches (e.g., creating a <code>feature-update</code> branch for the limb positioning logic), followed by secure merges back into the main branch.</li>
  <li><b>Remote Deployment:</b> The local history was successfully pushed to this GitHub repository.</li>
</ul>
